*ConfigMap*

ConfigMap.nvim	Quick loader for keymaps, commands, functions, and autocmds

NAME
	ConfigMap - lightweight declarative loader for Neovim config items

SYNOPSIS
        Use `:lua require('ConfigMap').setup(opts)` or call from your plugin
        manager's `config`.

DESCRIPTION
        `ConfigMap` accepts a single table (or a function that returns a
        table) and creates keymaps, user commands, autocmds, and validates
        simple named Lua functions. It's intentionally minimal and has no UI
        or external dependencies.

TOP-LEVEL FIELDS
	- `keymaps`  : list of keymap entries (see examples)
	- `commands` : list of user command entries
	- `funcs`    : list of named Lua functions (validated at setup)
	- `autocmds` : list of autocmd definitions
	- `defaults` : per-kind defaults merged into each item (optional)

EXAMPLES

SETTING KEYMAPS                                     *ConfigMap-keymaps*
    For more information about key mapping, see |usr_40.txt| and |40.1|.

    Keymaps are configured as tables with positional lhs and rhs, and optional
    desc and mode fields. The mode defaults to "n" if not specified.
        >lua

    Basic keymap:
            { '<leader>ff', ':Telescope find_files<CR>', desc = 'Find files' }

    Keymap with multiple modes:
            { '<leader>p', ':Telescope live_grep<CR>', desc = 'Live grep', mode = {'n','v'} }

    Keymap without description:
            { '<leader>q', ':q<CR>' }
<
SETTING HOTKEYS
	Always provide keymaps via the plugin-manager `opts = { ... }` table.
	Example Lazy.nvim spec:
		local spec = {
		  'jemjam/ConfigMap.nvim',
		  opts = {
			defaults = { keymaps = { silent = true, noremap = true } },
			keymaps = {
			  { '<leader>ff', ':Telescope find_files<CR>', desc = 'Find files' },
			  { '<leader>p', ':Telescope live_grep<CR>', desc = 'Live grep', mode = {'n','v'} },
			},
			commands = {
			  { name = 'SayHello', handler = function() print('hello') end, opts = { desc = 'Say hello' } },
			},
		  },
		}

NOTES
	- `setup` accepts either a table or a function returning the table.
	- Duplicate keymaps (same mode + lhs) or duplicate command/func names will raise an error.
	- This module does not expose functions globally â€” it only validates and registers
	  the provided definitions with Neovim.
