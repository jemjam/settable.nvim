*ConfigMap*

ConfigMap.nvim	Quick loader for keymaps, commands, functions, and autocmds

NAME
	ConfigMap - lightweight declarative loader for Neovim config items

SYNOPSIS
	Use `:lua require('ConfigMap').setup(opts)` or call from your plugin manager's `config`.

DESCRIPTION
	`ConfigMap` accepts a single table (or a function that returns a table) and
	creates keymaps, user commands, autocmds, and validates simple named Lua
	functions. It's intentionally minimal and has no UI or external dependencies.

TOP-LEVEL FIELDS
	- `keymaps`  : list of keymap entries (see examples)
	- `commands` : list of user command entries
	- `funcs`    : list of named Lua functions (validated at setup)
	- `autocmds` : list of autocmd definitions
	- `defaults` : per-kind defaults merged into each item (optional)

EXAMPLES

SETTING HOTKEYS
	Always provide keymaps via the plugin-manager `opts = { ... }` table and
	use the same array form as `vim.keymap.set`.
	Example Lazy.nvim spec:
		local spec = {
		  'jemjam/ConfigMap.nvim',
		  opts = {
			defaults = { keymaps = { silent = true, noremap = true } },
			keymaps = {
			  { 'n', '<leader>ff', ':Telescope find_files<CR>', { desc = 'Find files', silent = true } },
			  { {'n','v'}, '<leader>p', ':Telescope live_grep<CR>', { desc = 'Live grep' } },
			},
			commands = {
			  { name = 'SayHello', handler = function() print('hello') end, opts = { desc = 'Say hello' } },
			},
		  },
		  config = function(_, opts) require('ConfigMap').setup(opts) end,
		}

	This is the recommended and preferred usage pattern; direct `require(...).setup()` calls
	are discouraged to avoid duplication of plugin manager configuration.

NOTES
	- `setup` accepts either a table or a function returning the table.
	- Duplicate keymaps (same mode + lhs) or duplicate command/func names will raise an error.
	- This module does not expose functions globally â€” it only validates and registers
	  the provided definitions with Neovim.
