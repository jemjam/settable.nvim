*settable.nvim*	Simple helper for loading user commands, keymaps, etc

=============================================================================

*settable*

settable.nvim	Quick loader for keymaps, commands, and autocmds

NAME
	settable - lightweight declarative loader for Neovim config items

SYNOPSIS
        Use `:lua require('settable').setup(opts)` or call from your plugin
        manager's `config`.

DESCRIPTION
        `settable` accepts a single table (or a function that returns a
        table) and creates keymaps, user commands, and autocmds. It's
        intentionally minimal and has no UI or external dependencies.

TOP-LEVEL FIELDS
	- `keymaps`  : list of keymap entries (see examples)
	- `commands` : list of user command entries
	- `autocmds` : list of autocmd definitions (currently untested)

EXAMPLES

SETTING KEYMAPS                                         *settable-keymaps*
    For more information about key mapping, see |usr_40.txt| and |40.1|.

    Keymaps are configured as tables with positional lhs and rhs. You can also
    include desc and mode fields. The mode defaults to "n" if not specified.
>lua
    Basic keymap:
        { '<leader>ff', ':Telescope find_files<CR>', desc = 'Find files' }

    Keymap with multiple modes:
        { 
            '<leader>p', 
            ':Telescope live_grep<CR>', 
            desc = 'Live grep', 
            mode = {'n','v'} 
        }

    Keymap without description:
        { '<leader>q', ':q<CR>' }

SETTING HOTKEYS
	Always provide keymaps via the plugin-manager `opts = { ... }` table.
	Example Lazy.nvim spec:
		local spec = {
		  'jemjam/settable.nvim',
		  opts = {
			defaults = { keymaps = { silent = true, noremap = true } },
			keymaps = {
			  { '<leader>ff', ':Telescope find_files<CR>', desc = 'Find files' },
			  { '<leader>p', ':Telescope live_grep<CR>', desc = 'Live grep', mode = {'n','v'} },
			},
			commands = {
			  { ':SayHello', function() vim.notify('hello') end, opts = { desc = 'Say hello' } },
			},
		  },
		}

NOTES
	- `setup` accepts either a table or a function returning the table.
	- Duplicate keymaps (same mode + lhs) or duplicate command/func names will raise an error.

FILES
	- Lua module: `lua/settable/init.lua`
	- Types (EmmyLua): `lua/settable/types.lua`

SEE ALSO
	See the project README for additional examples.
